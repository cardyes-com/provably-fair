How to verify

roll = HMAC_SHA256(server_seed, client_seed + ":" + nonce) % 100000000

1

Compare sha256(server_seed) with the public hash.

2

Recalculate the roll from the revealed seed, client seed, and nonce.

3

Match the calculated roll against the value shown in the opening result.

Here's what each field means
client_seed

The client seed is randomly generated by default when you sign up. You can change it to any value you want at any time. The system will remember your last set client seed. You can verify it later using the Unboxing ID.

server_seed

The server seed is hidden by default. This prevents pre-simulation of results. Once the server seed has been used, you can choose to reveal it and verify that it has not been tampered with. A new hidden server seed is then provided for future openings.

public_hash

This value is generated by hashing the server seed. It is published with the roll data and can be used to verify that the server seed was not altered before being revealed.

nonce

This ensures that every opening result is independent and truly random.

roll

The roll is the result claimed by the system. This validator recalculates it from the revealed server seed, client seed, and nonce.
